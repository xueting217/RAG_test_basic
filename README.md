# RAG_test_basic
基于服装商品属性构建的基础 RAG（检索增强生成）项目，提供**本地知识库全生命周期管理**、向量存储、多轮会话记忆与 Web 可视化交互能力。用户可自主上传/更新私有服装商品知识，系统基于本地知识库完成精准问答，全程数据本地化存储，保障数据隐私。

## 核心流程
RAG 整体流程分为**离线知识库构建**与**在线智能问答**两大阶段，实现「私有知识入库-精准检索-智能生成」的全链路闭环：

### 一、离线流程：本地知识库构建（知识更新）
1. **文件加载**：读取本地服装商品相关知识文件（文本格式）；
2. **文本切分**：基于自然段落分割规则，将长文本切分为固定长度的文本块（支持重叠窗口，保障语义完整性）；
3. **去重校验**：通过 MD5 哈希值校验文本唯一性，避免重复入库；
4. **向量化存储**：将切分后的文本块通过嵌入模型向量化，存储至本地 Chroma 向量库，并记录元数据（来源文件、创建时间、操作人）。

### 二、在线流程：智能问答交互
1. **问题向量化**：将用户输入的服装商品相关问题转换为向量表示；
2. **相似性检索**：在向量库中检索与问题向量最相似的知识片段（Top-K 召回）；
3. **提示词组装**：将召回的知识片段、用户问题、会话历史整合为结构化提示词；
4. **智能生成**：提交提示词至大模型，基于本地私有知识生成精准回答。

## 项目结构与文件说明
### 一、目录结构
```
RAG_test_basic/
├── data/                # 私有知识文件存储目录（用户上传的服装商品知识文件）
├── chroma.db/           # Chroma 向量知识库（本地化存储向量化后的知识数据）
├── chat_history/        # 会话记忆存储目录（持久化用户与系统的多轮对话记录）
├── app_file_upload.py   # 知识库更新 Web 入口（Streamlit 实现）
├── knowledge_base.py    # 知识库核心服务（文本切分、去重、向量化入库）
├── app_qa.py            # 智能问答 Web 主程序（Streamlit 实现交互页面）
├── config_data.py       # 项目全局配置（向量库、文本切分、嵌入模型等参数）
├── file_history_store.py# 会话记忆管理服务（历史记录的增删改查）
├── rag.py               # RAG 核心逻辑（检索链、提示词组装、大模型调用）
└── vector_stores.py     # 向量检索服务（向量库初始化、相似性检索）
```

### 二、核心文件功能详解
| 文件名称                | 所属流程       | 核心功能                                                                 |
|-------------------------|----------------|--------------------------------------------------------------------------|
| `knowledge_base.py`     | 离线流程       | 1. 文本切分（递归分割+长度/重叠配置）；2. MD5 去重校验；3. 文本向量化并写入 Chroma 向量库；4. 元数据（来源/时间/操作人）记录。 |
| `app_file_upload.py`    | 离线流程       | Streamlit 网页服务，提供本地文件上传入口，调用 `knowledge_base.py` 完成知识库更新。 |
| `vector_stores.py`      | 在线流程       | 初始化 Chroma 向量库，封装向量检索接口，返回可接入 RAG 链的检索器实例。|
| `rag.py`                | 在线流程       | 集成向量检索、会话记忆、提示词模板，构建完整的 RAG 执行链，调用大模型生成回答。 |
| `app_qa.py`             | 在线流程       | Streamlit 交互页面，提供问答输入框、会话历史展示，调用 `rag.py` 完成问答交互。 |
| `file_history_store.py` | 在线流程       | 管理会话历史记录，支持多轮对话记忆的存储、读取与清理，保障上下文连贯性。 |
| `config_data.py`        | 全局配置       | 统一配置向量库路径、文本切分参数（chunk_size/chunk_overlap）、嵌入模型、MD5 记录文件路径等核心参数。 |

### 三、关键技术细节
1. **文本切分策略**：采用 `RecursiveCharacterTextSplitter` 递归分割，支持自定义分隔符、文本块长度、重叠窗口，适配服装商品长文本的语义完整性；
2. **去重机制**：基于文本内容 MD5 哈希值校验，避免重复知识入库，提升检索效率；
3. **向量存储**：使用 Chroma 本地向量库，轻量化部署，无需依赖云端服务，保障数据私有化；
4. **嵌入模型**：集成 DashScope Text-Embedding-V4 模型，适配中文服装商品文本的向量化表达；
5. **会话记忆**：本地化存储对话历史，支持多轮问答上下文关联，提升交互体验。

## 快速启动
### 1. 知识库更新
```bash
streamlit run app_file_upload.py
```
访问网页后，上传本地服装商品知识文件，系统自动完成文本切分、去重、向量化入库。

### 2. 智能问答
```bash
streamlit run app_qa.py
```
在交互页面输入服装商品相关问题，系统基于本地知识库生成精准回答，并展示会话历史。

## 核心特性
- 📁 **私有化部署**：所有知识文件、向量数据、会话记录均本地存储，无数据泄露风险；
- 🔍 **精准检索**：基于向量相似性检索，快速召回服装商品相关知识片段；
- 🚫 **智能去重**：MD5 哈希校验，避免重复知识入库，降低存储与检索成本；
- 💬 **会话记忆**：支持多轮对话上下文关联，提升连续问答体验；
- 🖥️ **可视化交互**：Streamlit 网页界面，无需编程基础即可完成知识库更新与问答交互。

## 测试设计
### 1. 测试目标
验证新智能查询模板在客服查询客户售后信息时，是否能显著提升以下指标：
- 关键信息提取准确率
- 员工查询效率
- 客户满意度

### 2. 测试指标
| 指标 | 定义 | 计算逻辑 | 采集方式 |
|------|------|----------|----------|
| **准确率** | 正确提取的关键信息项数占总项数的比例 | 准确率 = (正确提取项数 / 总关键信息项数) × 100% | 人工标注 + 系统自动比对 |
| **查询效率** | 完成一次查询的平均时间 | 平均耗时 = 总查询时间 / 查询次数 | 系统自动记录时间戳 |
| **客户满意度（CSAT）** | 客户对服务的满意程度 | CSAT = 所有客户评分的平均值（1-5分制） | 服务后自动推送满意度调查 |

### 3. 执行方案
- **测试对象**：50名客服 + 1000条查询记录
- **随机分组**：将50名客服随机分为两组，每组25人
  - 对照组：使用现有流程
  - 实验组：使用新智能查询模板
- **样本量**：每组处理500条查询记录，每位客服处理20条
- **测试周期**：7天（1天培训 + 6天正式测试）
- **监控机制**：每天收集数据，实时监控系统稳定性

### 4. 数据模拟
基于业务场景，模拟以下数据分布：
- **对照组**：准确率70%、查询耗时100秒、CSAT 4分
- **实验组**：准确率85%、查询耗时70秒、CSAT 5分

## 测试结果

| 指标 | 对照组 | 实验组 | 提升幅度 |

- **满意度**：t检验，p值 < 0.001，差异显著

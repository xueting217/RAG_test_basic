# RAG_test_basic
基于服装商品属性构建的基础 RAG 检索增强生成项目，包含完整的本地知识库构建、向量存储、会话记忆与 Web 交互功能。用户可自主更新私有知识，系统基于本地知识库进行精准问答。

RAG整体流程分为**离线流程**与**在线流程**两部分：

## 一、离线流程---更新本地知识库
1. 加载并读取本地知识文件
2. 对文本内容进行文本切分
3. 将切分后的文本向量化并存储至向量知识库

## 二、在线流程
1. 对用户输入的问题进行向量化处理
2. 与向量知识库进行向量相似度匹配，召回相关知识片段
3. 将匹配结果与用户问题整合组装为提示词
4. 提交给大模型，基于本地知识生成最终回答

# 项目文件与功能说明
## 一、文件存储说明
- `data`：用于存储私有知识文件。
- `chroma.db`：向量知识库，用于存储向量化后的知识数据。
- `chat_history`：用于存储用户与系统的长期会话记忆。

## 二、代码文件功能说明
### 1. 离线流程相关代码（知识库更新）
- `app_file_upload.py`：基于Streamlit框架开发的Web网页服务，作为知识库更新主程序，提供本地知识文件上传入口。
- `knowledge_base.py`：知识库更新服务，负责解析用户上传的文件并将其向量化，存储至向量知识库；同时通过字符串MD5值实现知识去重功能。
- 以上两个文件协同实现RAG离线流程。

### 2. 在线流程及核心配置代码
- `app_qa.py`：项目主程序，通过Streamlit启动Web交互页面，供用户进行对话交互。
- `config_data.py`：项目配置文件，用于配置项目相关参数。
- `file_history_store.py`：长期记忆存储服务，负责管理和存储会话历史记录。
- `rag.py`：RAG核心服务，用于获取执行链，集成向量检索、提示词模型及文件历史记录功能。
- `vector_stores.py`：向量存储服务，用于获取可加入执行链的检索器，并实现向量知识库的检索功能。
